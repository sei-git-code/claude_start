description: 変更をステージング・コミット・pushし、PRを作成する（Boris Cherny流ワークフロー）
---

現在のディレクトリのgit変更をコミットし、リモートにpushしてPRを作成する。以下の手順で進める。

1. `git status` と `git diff` で変更内容を確認する
2. 意味のある単位でファイルをステージングする（秘密情報を含むファイルを誤って含めていないか確認する）
3. 変更内容から「なぜ」を中心にした簡潔なコミットメッセージを作成し、コミットする
4. 現在のブランチが `main`/`master` の場合は、コミット前に作業用ブランチを新規作成してから進める（mainへの直接pushはしない）
5. リモートにpushする
6. `gh pr create` でPRを作成する（タイトルは簡潔に、本文にはSummaryとTest planを含める）
7. 作成したPRのURLを報告する

`gh` コマンドが認証されていない場合は、その旨を伝えて `gh auth login` を実行するようユーザーに促す（このコマンド内で自動的にブラウザ認証を完了させることはできない）。
